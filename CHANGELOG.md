# Changelog

Все заметные изменения в этом проекте будут описаны в данном файле по семантическому версионированию.

## Версия 5 (Текущая) - [28.04.2026]

### [5.3.0] - 2026-04-28
#### Добавлено (Doze-защита и Runtime Permissions Onboarding)
- **Doze Whitelist**: Системный in-app диалог `REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` при старте сервиса. Без whitelist'а Doze игнорирует `PARTIAL_WAKE_LOCK` и замораживает `delay()` в scanning loop (подтверждено на TECNO Pova 7 Ultra / HiOS 15, провалы до 37 минут).
- **OEM Autostart Guide**: Однократный AlertDialog на Transsion-устройствах (TECNO/Infinix/Itel) с инструкцией по включению автозапуска в Phone Master. Детекция через `Build.MANUFACTURER`.
- **`OemBatteryHelper.kt`** [NEW]: Утилита для детекции OEM, запроса battery whitelist и показа OEM-специфичных гайдов.
- **Runtime Permissions Onboarding**: Пошаговый запрос всех разрешений при первом запуске (как у Wildberries):
  1. 📍 Геолокация (`ACCESS_FINE_LOCATION` + `ACCESS_COARSE_LOCATION`)
  2. 🏃 Физическая активность (`ACTIVITY_RECOGNITION`)
  3. 🔔 Уведомления (`POST_NOTIFICATIONS`, Android 13+)
  4. 📍 Фоновая геолокация (`ACCESS_BACKGROUND_LOCATION`)
  5. 🔋 Исключение из Doze (`REQUEST_IGNORE_BATTERY_OPTIMIZATIONS`)
- **Диагностика**: Поле `battWhitelist=yes/no` в событии `SERVICE_START`.

#### Изменено (Адаптивная вёрстка контролёра)
- **Компактный layout**: Уменьшены padding (16→12dp), margins, высота кнопок (56→44dp, 48→40dp), размеры шрифтов для комфортного отображения на экранах <6".
- **4 колонки этажей** вместо 3 — помещается больше информации без прокрутки.
- **Явный padding** на всех кнопках — текст не упирается в края.
- Лог: заголовок «Лог чекпоинтов» → «Лог», шрифт 12→11sp.

#### Удалено
- **«Сохранить прогресс (JSON)»**: Кнопка, лаунчер и функция `saveJsonToUri()` полностью удалены из TasksFragment. Данные сохраняются через CSV + Yandex Disk.

### [5.2.1] - 2026-04-22
#### Исправлено (Критическое: ANR-крэши при остановке сервиса)
- **Неблокирующий flush()**: Заменён `runBlocking { doUpload() }` в `IncrementalSyncer.flush()` на `UploadQueueManager.enqueue()`. Метод `flush()` теперь мгновенный — добавляет файл в персистентную JSON-очередь вместо блокирования Main Thread сетевым вызовом. Устраняет ANR при отсутствии интернета.
- **Устранение Race Condition**: Убран `stopSelf()` в `handleScanResults()` для task-режима. Управление жизненным циклом сервиса в task-режиме делегировано исключительно `TasksFragment.observeScanning()`. Предотвращает двойной stop из IO и Main Thread одновременно.
- **Порядок shutdown**: `serviceScope.cancel()` перенесён в начало `onDestroy()` (до `flush()`). Scanning loop гасится сразу — нет «призрачных» SCAN_REQUEST после начала shutdown. Событие `SERVICE_STOP` теперь записывается в диагностику.

### [5.1.0] - 2026-04-22
#### Добавлено (Инкрементальная синхронизация и Offline-устойчивость)
- **`IncrementalSyncer.kt`** [NEW]: Периодический re-upload CSV на Яндекс.Диск каждые ~30 сек (6 снимков × 5 сек) прямо во время сканирования. Watchdog 60 сек для зависших uploads. Автоматическое создание удалённых папок.
- **`UploadQueueManager.kt`** [NEW]: Персистентная JSON-очередь загрузки (`upload_queue.json`). Файлы копятся при отсутствии сети, загружаются при восстановлении. Ретрай каждые 5 минут, до 10 попыток. Watchdog 5 минут для зависших задач.
- **Параллельная синхронизация диагностики**: Второй `IncrementalSyncer` (`diagSyncer`) для файла `diag_*.csv`, работает параллельно с основным CSV-синхронизатором.
- **Маршрутизация диагностики**: Диагностические файлы загружаются в разные папки в зависимости от режима скана (`diag_scans/` vs `diag_tasks/`).

#### Исправлено (Целостность данных)
- **Потеря последнего снимка**: `stopService()` теперь вызывается ДО `stopSession()` с задержкой 500ms, гарантируя, что последний IO-корутин успевает завершить инкремент счётчиков.

### [5.0.0] - 2026-04-18
#### Добавлено (Диагностика и мониторинг WiFi)
- **`WifiStateMonitor.kt`** [NEW]: Реалтайм-мониторинг состояния WiFi через BroadcastReceiver. Логирование событий enabled/disabled/enabling/disabling в диагностику.
- **WiFi BroadcastReceiver в сервисе**: Дублирующий `wifiStateReceiver` непосредственно в `WifiScanService` для отслеживания WiFi-событий при выключенном экране (когда фрагменты уничтожены).
- **Расширенный DiagnosticLogger**: Новые события: `WIFI_STATE`, `SYNC_INIT`, `SYNC_OK`, `SYNC_FAIL`, `SYNC_FLUSH`, `SYNC_WATCHDOG`, `SYNC_ERROR`.
- **`ControllerCsvLogger.kt`** [NEW]: Утилита для логирования контроллерных данных.

## Версия 4 (Предыдущая) - [09.04.2026]

### [3.0.1] - 2026-04-09
#### Добавлено (UX и Обратная связь)
- **Контекстное меню (`Action Sheet`)**: Стандартное выпадающее меню заменено на современный всплывающий снизу Action Sheet (`BottomSheetDialog`) для управления локациями и вызова настроек.
- **Тактильный и звуковой отклик**: Добавлена микро-вибрация и системный звук уведомления при успешном достижении лимита сканирования для локации (5/5). Отклик считывает системный профиль (без звука / вибро / звонок) для защиты от мешающего шума.
- **Унификация UI**: Вынос программного интерфейса в жесткие `XML`-компоненты (`res/layout/component_*`). Высота пунктов меню Action Sheet выведена в комфортные 44dp, кнопка Отмена использует классический компонентный `MaterialButton`. В настройках убран ложный отступ текста (отключено резервирование места под иконки).

### [3.0.0] - 2026-04-05
#### Добавлено (Сенсоры и Данные)
- **Номер канала Wi-Fi (`Channel`)**: Мобилка конвертирует частоту в номер канала (2.4/5/6 GHz) перед записью в CSV. Компактнее для хранения и передачи.
- **Сырой Timestamp сети (`NetworkTimestamp`)**: Индивидуальное время фиксации каждой Wi-Fi сети радиомодулем (wall-clock), приведено к формату `yyyy-MM-dd HH:mm:ss` (без миллисекунд), чтобы стандартизировать парсинг на Python.
- **Улучшенная точность GPS (`Latitude`, `Longitude`)**: Точность координат в CSV и UI повышена до 7 знаков после запятой, принудительная локаль `US` (всегда точка вместо запятой).
- **IMU/PDR сенсоры**: Локальная обработка данных акселерометра, шагомера и гироскопа на устройстве.
  - `StepsDelta` — шаги за последний интервал сканирования.
  - `StepsTotal` — общий счётчик шагов с начала сессии.
  - `Azimuth` — направление движения (Sensor Fusion из гироскопа).
  - `AzimuthConfidence` — достоверность азимута (0.0-1.0).
  - `DeviceOrientation` — ориентация устройства, строго записанная профессиональными константами для машинного обучения, без локализации: `PORTRAIT`, `REVERSE_PORTRAIT`, `LANDSCAPE`, `REVERSE_LANDSCAPE`, `FACE_UP`, `FACE_DOWN`, `TILTED`. (Эти теги помогают компенсировать эффект «антенна вниз» и пробивания перекрытий).
- Новый модуль `SensorCollector.kt` — Edge Computing: сырые IMU-данные обрабатываются на устройстве, в CSV пишутся только агрегированные результаты.
- Новая утилита `FrequencyToChannel.kt`.
- **Программное охлаждение (Warmup/Cooldown)**: Внедрена настраиваемая задержка (`pref_scan_cooldown`, по умолчанию 5 секунд) перед созданием первого слепка сессий. Защищает датасет от "грязных стартовых сканов".

#### Добавлено (Редизайн UI и UX Заданий)
- Умная система вытеснения интерфейса (Displacement Navigation): на нижнем баре отсутствует кнопка создания новых локаций, она гармонично внедрена в `[ + ]` в верхнем меню Breadcrumbs. Кнопка `[⬇ СЛЕДУЮЩИЙ ЭТАЖ]` всплывает снизу из-за границ экрана исключительно после успешного завершения всего этажа.
- **Глобальная защита "от дурака" (Smart Locking)**: Мгновенное скрытие кнопок `[Скан]` у остальных дочерних элементов, пока любая локация на этаже находится в статусе записи. Предотвращает случайные коллизии потоков.
- **Data-Science Grade Interface**: Полный редизайн карточек заданий. Интегрированы объемные `Material Cards` (тень 4dp, радиус 8dp, ripple-фон), аналогичные карточкам из "Истории сессий".
- Динамическая высота локации зафиксирована через `minLines="2"`, что математически полностью устранило эффект "прыгающих кнопок" при начале записи.
- Увеличение площади и веса основных действий на карточках — переход на `MaterialButton` высотой 56dp.
- Внедрение классических интуитивно-понятных векторных иконок для действий (`Play` для скана, залитая `Stop` для отмены сессии).
- Внедрено `RightValuePreference` для выравнивания значений в настройках.

#### Обратная совместимость
- Все 9 новых столбцов добавлены **в конец** CSV. Старые столбцы не изменены и не удалены.
- Python-бэкенд (`pandas.read_csv`) продолжает работать без изменений.
- Graceful Degradation: если сенсор отсутствует на устройстве, соответствующие столбцы записываются пустыми.

## Версия 3 (Предыдущая) - [20.03.2026]

### [2.0.0] - 2026-03-20
#### Добавлено
- Вкладка "Задания" (Tasks) для динамического обхода по многоуровневой иерархии (Синхронизация JSON через SAF).
- Умный авто-переход ("Вперед" стрелкой) между этажами при 100% выполнении.
- Автогенерация уникальных имен единого мастера CSV-файла `wifi_tasks_blueprint_YYYYMMDD_HHmm.csv`.
- Экспорт измененного дерева счетов (JSON) в PrettyPrint-формате для питона.

#### Исправлено
- Критический баг зависания таймера "3/5" и "двойных сканов в Истории": переменная `activeScanNode` перенесена в глобальный `WifiRepository` для надежного выживания при уничтожении UI (ViewPager2 Tab Switching).
- Служба `WifiScanService` полностью автономна: читает `EXTRA_REQUIRED_SCANS` и выключается без подсказок фрагмента.
- Иерархические переменные (Address, Floor) пробрасываются напрямую в Intent фоновой службы.
- Баг с датой 1913 года (микросекунды System Boot вместо UNIX Timestamp) при скане сетей — заменен на жесткий `Batch Time` (единый Date() для всей пачки сетей).
