# Контекст мобильного приложения (WifiScanner)

## Версия 3 (Текущая) - [20.03.2026]
### 1. Стек технологий
- **Язык**: Kotlin
- **Архитектура**: MVVM + Singleton State (WifiRepository) + Foreground Service + Android SAF
- **Платформа**: Android (Smart Tasks Blueprint)

### 2. Принцип работы (Smart Tasks Blueprint v2.0)
Приложение полностью переведено с ручного сканирования на обход по загружаемой цифровой карте (JSON Blueprint). 
1. **Загрузка Карты**: Через SAF (Storage Access Framework) читается многоуровневая структура заданий (Объект -> Адрес -> Подъезд -> Этаж -> Локация).
2. **Фоновое автономное сканирование**: При старте задания поднимается автономный `WifiScanService`. Он сам знает лимит слепков (например, 5), собирает данные (опрашивая Android Wi-Fi Chip + FusedLocationProvider для снятия системных лимитов), и самостоятельно останавливается с записью в Историю.
3. **Хранение состояния (Жизненный цикл)**: Классическая проблема уничтожения ViewPager2-фрагментов (TasksFragment) решена выносом `rootNode`, `navStack`, и самое главное — `activeScanNode` — в глобальный синглтон `WifiRepository.kt`. Это позволяет UI безопасно отключаться и подключаться к запущенному сервису.

### 3. Формат экспорта данных (Flat-Table Pipeline)
- Единый **плоский CSV-файл** (Flat-Table) на всю сессию обхода здания вместо дробления на 50 файлов.
- Человекочитаемые столбцы: `Timestamp; NodeId; Address; Entrance; Floor; LocationName; SSID; MAC; RSSI; Frequency; RecordNumber`.
- Исправление микросекундных разбросов: вычисляется единый `Batch Time` для всех сетей внутри одной пачки (слепка) — идеальная синхронность. Физическое вырезание ("Очистить") прямо из единого CSV по `NodeId`.
- Экспорт обновленного JSON (`setPrettyPrinting`) с тегами прогресса на бэкенд.

## Версия 2 (Предыдущая) - [18.03.2026]
### 1. Стек технологий
- **Язык**: Kotlin
- **Архитектура**: MVVM (Model-View-ViewModel)
- **Платформа**: Android

### 2. Основной функционал (MVP)
1. **Фоновое сканирование**: Приложение запрашивает уровни сигналов (RSSI) и идентификаторы точек доступа (MAC).
2. **Экспорт (Data Persistence)**: Результаты сохраняются локально в формате CSV-файла (Каждая точка — отдельный файл).
3. **Отображение (UI)**: Вывод в UI (список сетей, ViewPager2: "Текущие сети" и "Запись").

### 3. Правила разработки (Constraints)
- Архитектура: Всю логику взаимодействия с WifiManager инкапсулировать внутри WifiViewModel. Activity работает только с ViewPager. 
- Обход системных ограничений (Wi-Fi Scan Throttling): FusedLocationProvider + Loop + Foreground Service. Требуются `ACCESS_BACKGROUND_LOCATION`.
