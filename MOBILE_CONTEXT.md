# Контекст мобильного приложения (WifiScanner)

## 1. Стек технологий
- **Язык**: Kotlin
- **Архитектура**: MVVM (Model-View-ViewModel) + LiveData / StateFlow
- **Платформа**: Android (минимальный SDK и Target SDK согласно build.gradle)

## 2. Основной функционал (MVP)
1. **Фоновое сканирование (Background Scanning)**: Приложение запрашивает у системного Wi-Fi Manager уровни сигналов (RSSI) и идентификаторы точек доступа (BSSID/MAC).
2. **Экспорт (Data Persistence)**: Результаты сканирования сохраняются локально в формате CSV-файла для последующего использования в Python-скриптах.
3. **Отображение (UI)**: Вывод результатов в UI (список сетей в реальном времени, отображение MAC-адреса и RSSI в dBm). Приложение разделено на 2 логические вкладки с помощью `ViewPager2`: "Текущие сети" (просмотр) и "Запись" (управление сканированием).

## 3. Правила разработки (Constraints)
- **UI:** Для верстки использовать строго `XML Layouts` (уже есть `activity_main.xml`, `item_wifi.xml`, `fragment_view.xml`, `fragment_scan.xml`). Не использовать Jetpack Compose.
- **Архитектура:** Всю логику взаимодействия с `WifiManager` инкапсулировать на слое репозитория или внутри `WifiViewModel`. Activity должна заниматься только отображением вкладок (`ViewPager2` + `TabLayout`) и запросом прав. Логика разделена по фрагментам: `ViewFragment` и `ScanFragment`.
- **Ограничения платформы:** Для обхода жесткого троттлинга Wi-Fi сканирования (Wi-Fi Scan Throttling) в Android используется гибридный подход: параллельно с `WifiManager` мы запускаем высокочастотный опрос `LocationManager`, получая пассивные обновления сканирования эфира самой ОС. Это позволяет собирать плотные данные без изменения настроек разработчика. Требуются runtime-доступы: `ACCESS_FINE_LOCATION`, `ACCESS_COARSE_LOCATION`.

## 4. Основные сущности
- `WifiScanResult`: Дата-класс, описывающий одну найденную сеть в единицу времени.
- `ScanSession`: Дата-класс истории сканирования (Имя, Время старта/стопа, Количество слепков и записей).
- `WifiViewModel`: Главный узел бизнес-логики UI-слоя, используемый для вкладки Записи.
- `MainActivity`: Хост для вкладок. Запрашивает только **базовые разрешения** (`ACCESS_FINE_LOCATION`, `ACCESS_COARSE_LOCATION`) при старте. Если разрешения не выданы — показывает UI-заглушку.
- `ViewFragment`: Фрагмент для отображения найденных сетей. Работает автономно: циклично получает `WifiManager.scanResults` каждые 2 секунды без привязки к фоновой службе.
- `ScanFragment`: Фрагмент для управления фоновым сервисом. Перед запуском сервиса самостоятельно запрашивает **расширенные права** (`ACCESS_BACKGROUND_LOCATION`, `POST_NOTIFICATIONS`) с предварительным объяснением причины (лучший UX).
- `PermissionHelper`: Утилита, разделяющая списки базовых (foreground) и фоновых (background) разрешений.
