# Контекст мобильного приложения (WifiScanner)

## Версия 3 (Текущая) - [05.04.2026] (Smart Tasks UX & Sensor UI Polish)
### 1. Архитектура и UI (Data-Science Grade Interface)
- Изменен подход к интерфейсу "Заданий" (Smart Tasks): Плоские списки заменены на **объемные Material Cards** (`androidx.cardview.widget.CardView` с радиусом 8dp и тенью 4dp), обеспечивая полное соответствие "Истории сессий".
- **Динамическая Навигация (Displacement)**: Исключены визуальные коллизии снизу экрана. Кнопка новой локации перенесена в Breadcrumbs верхнего бара (`[ + ]`). Кнопка `[⬇ СЛЕДУЮЩИЙ ЭТАЖ]` всплывает снизу из пустоты только после завершения всех текущих локаций на этаже.
- **Стабилизация верстки**: Строка статуса локации зафиксирована через `android:minLines="2"`. Это полностью устраняет эффект "прыгающей кнопки" при старте/остановке сканирования (высота карточки абсолютно стабильна).
- **Harmony & Weight (Кнопки)**: Кнопки действий переведены на `MaterialButton`, получили высоту 56dp (полноразмерные) и интуитивные векторные иконки (`Play` и `Stop`).
- **Синхронизация состояний (Smart Locking)**: Службы `TasksFragment` и `TaskNodeAdapter` динамически сверяются со статусами сканирования. При старте любой локации, кнопки Скан у соседних мгновенно прячутся для предотвращения аномалий параллельного сбора.

### 2. Принцип работы (Включая Anti-Throttling и IMU Sensor Fusion)
Приложение выполняет обход по загружаемой цифровой карте (JSON Blueprint) с параллельным агрессивным сбором данных IMU-сенсоров (частота SENSOR_DELAY_UI). Наблюдение за потоком данных выведено в 3-колоночный UI Dashboard (Шаги | IMU | GPS), работающий через StateFlow.
1. **Архитектура Wi-Fi Сканирования (Anti-Throttling)**: Внедрена 100% защита от жестких блокировок ОС (Doze Mode и сканирование 4 раза в 2 минуты):
   - **Foreground Service**: Служба переднего плана типов `TYPE_LOCATION` и `TYPE_HEALTH` (Android 14+), чтобы не "засыпали" шагомер и GPS.
   - **Троянский конь (FusedLocationProvider)**: Прямые вызовы `wifiManager.startScan()` запрещены (приведут к блокировке сканера на 2 минуты). Вместо этого служба раз в 5 секунд запрашивает локацию `PRIORITY_HIGH_ACCURACY` у Google Play Services. Высшие привилегии Google аппаратно принуждают включиться Wi-Fi антенну.
   - **Пассивный Polling**: Приложение бесконечно считывает системный кэш `wifiManager.scanResults` куда "нечаянно" только что просканировал среды Google.
2. **IMU/PDR Edge Computing**: `SensorCollector.kt` обрабатывает сырые данные локально:
   - Шагомер: работает связка `STEP_COUNTER` + fallback на мгновенный `STEP_DETECTOR`.
   - Ориентация устройства: строго классифицируется на `PORTRAIT`, `REVERSE_PORTRAIT`, `LANDSCAPE`, `REVERSE_LANDSCAPE`, `FACE_UP`, `FACE_DOWN`, `TILTED` для компенсации эффекта «антенна вниз» на бэкенде.
3. **Логирование и Точность (Data Science Grade)**: 
   - **Философия "Пиши всё, фильтруй потом"**: Приложение намеренно собирает максимум сырых данных с радиоэфира, отдавая ответственность за отбраковку "призраков" (Ghost networks, зависших в кэше ОС) на откуп Python-бэкенду. Это гарантирует непрерывность записи IMU-данных и не позволяет приложению зависать при агрессивном троттлинге.
   - **Анти-Дубликаты**: При этом полностью идентичные слепки эфира (когда ОС вообще не сканировала, а вернула 100% старый кэш) отбраковываются по правилу: `if (currentMaxTimestamp == lastMaxTimestamp) { return }`. Счетчик прогресса сканирования (1/5) растет только при реальных обновлениях эфира.
   - `NetworkTimestamp` синхронизируется на `yyyy-MM-dd HH:mm:ss`. Этот параметр используется Python-скриптом для мгновенной отбраковки старых сетей (`df = df[df['Age'] < 7]`).
   - GPS-координаты пишутся с точностью до **7 знаков после запятой**, строго с разделителем `.` (`Locale.US`).
   - Упорядоченный `RecordNumber`: список аппаратно сортируется по RSSI *до* записи в CSV.
4. **Защита "Грязного старта" (Cooldown)**: Аппаратно внедрена задержка старта (по умолчанию 5 сек, читается из `pref_scan_cooldown`), дающая оператору стабилизировать телефон в пространстве и выжигающая кэш-призраки прошлых помещений.

### 3. Формат экспорта данных (Flat-Table CSV v3.1)
```
Timestamp; NodeId; Address; Entrance; Floor; LocationName; SSID; MAC; RSSI; Frequency; RecordNumber;
Channel; NetworkTimestamp; Latitude; Longitude; StepsDelta; StepsTotal; Azimuth; AzimuthConfidence; DeviceOrientation
```

## Версия 2 (Предыдущая) - [04.04.2026] (Итерация 4.0.0 prep)

- **Язык**: Kotlin (DLS build.gradle.kts)
- **Архитектура**: MVVM + Singleton State (WifiRepository) + Foreground Service (Health + Location) + IMU Sensor Fusion

### 2. Принцип работы (Smart Tasks Blueprint v2.0 + Sensor Data)
Приложение выполняет обход по цифровой карте с параллельным агрессивным сбором данных IMU-сенсоров.
1. **Wi-Fi Сканирование**: Автономный `WifiScanService` с `FusedLocationProvider` (Priority High Accuracy) для принудительного обновления системного кэша и обхода Android Throttling.
